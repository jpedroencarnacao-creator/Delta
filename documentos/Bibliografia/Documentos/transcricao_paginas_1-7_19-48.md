Deformacao do lumen.

Deformacao da via aerea durante a respiracao e expiracao.

Inspiracao:

- Lumen aumenta.
- A passagem fica com mais facilidade.

Expiracao:

- Lumen diminui.
- A passagem fica mais estreita.

Menor relevancia:

- Simulacao localizada da estenose traqueal ou bronquica, isto e, estreitamento
  anormal da via.


- Simulacao de diametro traqueal aumentado (traqueomalacia) ou diametro bronquico
  aumentado (bronquiectasias).

Formula e notas para a cinematica:

Z = z - servo_z_offset.

Retiramos o valor de offset da base ate ao centro do servo, para trabalharmos so com os valores de Z
das juntas.

Arm_end = X + L3.

Calcula o deslocamento da uniao na plataforma, somando o valor atual de X com a distancia L3.

L2p = sqrt(pow(L2, 2) - pow(y, 2)).

L2p = sqrt(L2^2 - y^2).

Calculada a projecao de L2 no plano XY.

L2Angle = asin(y / L2).

if(!(abs(L2Angle) < 35 graus)) return false.

Calcula o angulo entre L2 e a projecao L2p e verifica se nao passa de um angulo que seja impossivel

ext = sqrt(pow(z, 2) + pow(servo_offset_x - Arm_end_x, 2)).

Calcula a distancia em linha reta desde o eixo do servo ate a ultima junta na plataforma.

if(ext <= L2p - L1 || ext >= L1 + L2p) return false.

Verifica se a soma dos dois lados do triangulo (catetos) sao maiores que a hipotenusa (ext), para que
o triangulo seja possivel realizar.

phi = acos((pow(L1, 2) + pow(ext, 2) - pow(L2p, 2)) / (2 * L1 * ext)).

omega = atan2(z, servo_offset_x - Arm_end_x).

theta = phi + omega.

Calculo dos angulos junto do eixo do servo motor.

--- Pagina 40 ---

Pode confirmar o tempo de demora do ciclo atual sem a maquina de estados, so por volta dos 1 - 10 ms,
mesmo funcionando tudo em simultaneo.

Por agora o funcionamento completo com a fusao dos pontos ficou de uma forma muito parecida com a
estrutura antes estabelecida.

O indice de cada ponto de movimento e calculado conforme o tempo, permitindo uma maior configuracao
no movimento.

A partir do indice atual e o seguinte sao enviados para a funcao Intermed_Position que, em funcao do
tempo entre o ponto atual e o seguinte, vai calcular varios pontos para suavizar o movimento. Esta
abordagem permite uma acao rapida, precisa e suave no movimento e nao faz engasgar o sistema e os
servos com centenas de pontos desnecessarios.


--- Pagina 41 ---

Funcao Intermed_Position()

MotionStorage & m = *(Inst.def)

E chamada apenas a definicao e endereco do MotionStorage atraves do MotionInstance.
Isto permite que existam centenas de pontos nos arrays do storage e ele carregue o valor de forma mais
leve.

Condicao indicada:

if (!Inst.Active || m.nPoints <= 1 || m.Period <= 0 || Start == 0) {
    x = 0;
    y = 0;
    z = 0;
}

Isto faz com que em caso de alguma destas opcoes ser verdadeira:
- movimento desativado;
- nPoints ser menor ou igual a 1;
- Period ser igual a 0;
- Start estar desativado;

o valor de x, y e z seja 0.


--- Pagina 42 ---

Isto permite simplificar e criar redundancias no codigo.

Notas:
- elapsed -> decorrido.
- resp -> interpolacao linear.

Exemplo/estrutura:

int i = inst.iIndex;
int j = inst.jIndex;

if (Direction == 1) {
    elapsed = inst.elapsed_1;
}

if (Direction == -1) {
    elapsed = inst.elapsed_3;
}

Isto copia os valores de indice atual e futuro, bem como do tempo decorrido entre os pontos
(caso no sentido direto ou reverso).

alpha = elapsed / dtMs

em que:
- elapsed -> tempo decorrido;
- dtMs -> tempo por ponto.

if (alpha < 0) alpha = 0;
if (alpha > 1) alpha = 1;

Isto calcula alpha e define limites para que fique contido entre 0 e 1.


--- Pagina 43 ---

Permite definir 0 como o ponto atual, 1 como ponto seguinte e alpha como percentagem em funcao do
tempo.

x = lerp(x[i], x[j], alpha)
y = lerp(y[i], y[j], alpha)
z = lerp(z[i], z[j], alpha)

lerp -> A + (B - A) x t

No esquema:
- A corresponde ao ponto i;
- B corresponde ao ponto j;
- t corresponde a alpha.

Com isto transferimos o valor de alpha com um peso e calculamos o valor intermedio entre x atual e x
futuro em funcao do tempo. Depois e repetida a mesma logica para y e z.
